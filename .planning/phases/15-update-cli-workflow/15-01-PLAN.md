  <action>
创建 scripts/yongle-update.js，实现以下完整逻辑（per D-01 到 D-14）：

1. **ANSI 颜色常量与 Banner**：复用 install.js 中的 cyan/green/yellow/red/bold/dim/reset 常量和永乐大典 ASCII Banner 格式。

2. **安装模式检测**：检查 packageRoot 目录是否存在 `.git` 目录来判断 Git 模式 vs npm 模式（per D-01）。

3. **版本检测——双轨制**（per D-01, D-02）：
   - Git 模式：execSync `git fetch origin` 然后 `git rev-list HEAD..origin/main --count` 获取落后 commit 数。如果 count > 0 则有更新可用。
   - npm 模式：execSync `npm view yongle-dadian version` 获取最新版本号，与本地 package.json 的 version 字段比较。

4. **回滚准备**（per D-06）：Git 模式下在 pull 前通过 `git rev-parse HEAD` 记录当前 commit hash。

5. **拉取更新**（per D-03, D-05）：
   - Git 模式：execSync `git pull origin main`。
   - npm 模式：execSync `npm install -g yongle-dadian@latest`。

6. **智能依赖检测**（per D-04）：Git 模式下 pull 前计算 `package-lock.json` 的 MD5 hash（使用 crypto 模块），pull 后再次计算，仅在 hash 不同时执行 `npm install`。

7. **回滚处理**（per D-06, D-07）：任何步骤失败时，Git 模式下执行 `git reset --hard <原始hash>`。重注入失败时不回滚代码，只提示用户手动执行 install.js。

8. **重注入**（per D-08, D-09）：扫描已知运行时目录（~/.gemini/config/plugins/yongle-plugin/, ~/.claude/skills/, ~/.gemini/antigravity/skills/ 等）下的 `install-manifest.json` 文件，提取 runtime 和 scope 字段，对每个执行 `node <packageRoot>/bin/install.js --<runtime> --<scope>`。若未找到任何 manifest，默认执行 `--antigravity --global`。

9. **日志输出**（per D-10, D-11, D-12）：4 个步骤带编号 [1/4]、[2/4]、[3/4]、[4/4]，使用 ▸ 进行中、✓ 完成、✗ 失败。最终输出版本升级 Banner 和重启提示。

脚本入口使用 `#!/usr/bin/env node`，导出为 CommonJS 模块，可通过 `node scripts/yongle-update.js` 直接调用。
  </action>
